#

1. 确定需求
   - 需求总表定位需求，确定排期，提测时间，相关人员
   - 读prd

2. 拉群（如果还没有群）
   - 网关、产品、后台、ued、测试、ios
   - 群名称按需求指定
   - 群公告列出网关文档、ued文档、prd文档、测试用例文档、文案文档

3. 理思路
    - PRD需求
    - UED设计稿
    - 网关接口、数据
        - 读网关文档
        - 数据跑通
        - 新老版本数据兼容性
        - 考虑数据合理性、理清数据疑点
    - 新老版本兼容性
    - 外部团队对接（非研究院团队）
        - 研发、测试、验证

4. 开发
    - 了解关联模块
    - 设计代码，实施开发
        - 拉分支
        - 更改组件版本、版本号隔离

5. 验证
    - 测试数据获取
    - 自测模拟数据验证
    - 过测试用例文档（测试提供）
    - 埋点自测验证

6. 提测
    - 分支合并
    - Jira问题修复
    - Fabric问题修复
    - 自动化测试问题修复
    - 内存泄漏优化

7. 验收
    - UED验收（视觉还原）
    - 产品验收（业务功能、埋点）
    - 测试验收（需求功能、异常兜底、回归测试、文案统一）

8. 上线后监控
    - Fabric问题修复
